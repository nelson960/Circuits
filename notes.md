# Notes

This file is a plain-language map of the main concepts in this repo.

It is not a full theory document. It is a rough guide to:

- what the model is
- what training is doing
- what happens at inference time
- what the main internal objects are
- what our analysis tools are trying to measure

## Big Picture

We train a small decoder-only transformer on a synthetic next-token task.

The task is not the final scientific goal.

The real goal is to study:

- how a useful circuit appears during training
- which parts of the model become important first
- why gradient descent strengthens one mechanism instead of another

So we care about both:

- behavior: does the model solve the task?
- mechanism: how is the task being solved inside the network?

## Neural Network

A neural network is a function with many adjustable numbers called **weights**.

Those weights change during training.

The network takes an input sequence of tokens and produces logits for the next token.

In this repo, the network is a small **decoder-only transformer**.

## Training

Training means:

1. run the model forward on data
2. compare its prediction to the correct next token
3. compute a loss
4. compute gradients of that loss with respect to the weights
5. update the weights a little
6. repeat many times

This is standard gradient-based learning.

## Inference

Inference is just the forward pass:

1. read the tokens
2. compute hidden states layer by layer
3. produce logits for the next token
4. choose a prediction

No weights are changed during inference.

## Backpropagation

Backpropagation is the procedure that tells us:

- if the prediction was wrong,
- which weights contributed to that error,
- and in which direction they should move to reduce the error

It gives us **gradients**.

## Gradient Descent

Gradient descent means updating the weights in the direction that reduces loss.

Very roughly:

- bad prediction -> loss
- loss -> gradient
- gradient -> weight update
- updated weights -> slightly different computation next time

Our scientific question is about this loop:

- how do many small updates gradually create a circuit?
- why does one partial mechanism get reinforced instead of another?

## Checkpoint

A checkpoint is a saved copy of the model during training.

For this project, checkpoints are important because they let us ask:

- what could the model do at step `t`?
- what information existed at step `t`?
- what changed by step `t + 1` or `t + 250`?

We do not just study the final model. We study the path.

## Token

A token is one discrete item the model reads or predicts.

In this repo, tokens are synthetic symbols like keys, values, reads, and writes.

## Logits

Logits are the raw scores the model gives to every possible next token before softmax.

Higher logit means the model prefers that token more.

## Latent Program

The dataset is generated by an explicit underlying rule system.

That hidden rule system is the **latent program**.

It tells us what the correct answer is and helps us define:

- oracle behavior
- counterfactual tests
- OOD splits

## Weight

A weight is one learned number inside the model.

There are many weights.

Most weights change during training, but not all changed weights are equally important for the final mechanism.

## Neuron

In this repo, “neuron” usually means one hidden unit inside an MLP block.

A neuron has:

- an input direction
- an activation
- an output direction

Neurons are useful, but they are not always the best unit of explanation because transformer computations are often distributed.

## Head

A head is one attention head inside an attention layer.

Very roughly, a head can:

- look for something relevant
- attend to a source position
- copy or route information
- write something into the residual stream

Heads are often easier to track than single neurons.

## MLP Block

An MLP block is the feed-forward submodule inside a transformer block.

Very roughly, it can:

- transform the current representation
- write useful features into the residual stream
- combine information that earlier heads made available

## Transformer Block

Each transformer block contains:

- attention
- an MLP
- residual connections
- normalization

The model applies several of these blocks in sequence.

## Residual Stream

The residual stream is the main shared state that moves through the model.

It is one of the most important objects in transformer interpretability.

Heads and MLPs read from it and write back into it.

If you want to know where information lives in the model, the residual stream is often the first place to look.

## Residual Stage

A residual stage is one named point in the model where we record the residual stream, for example:

- after attention
- after the MLP
- after the final normalization

Examples in this repo:

- `layer_0_post_attn`
- `layer_2_post_mlp`
- `final_norm`

## Latent Space

A latent space is just some internal vector space where the model represents information.

The residual stream is one latent space.

An SAE feature basis is another way of describing structure inside that space.

## Feature

In the current shared-feature tools, a **feature** is one latent unit in a shared sparse autoencoder basis fitted to activations at a stage.

So a feature is:

- not a raw neuron
- not a head
- not automatically a clean semantic concept

It is better to think of a feature as:

- one direction or pattern in activation space that the analysis tool found useful

## Feature Family

A feature family is a group of features with similar trajectories over training.

They are grouped because they evolve similarly across checkpoints.

A family is not yet a proved mechanistic object.

It is a candidate submechanism:

- several features that seem to rise, stabilize, or drift together

## Circuit

A circuit is a causally important set of components that together implement part of the behavior.

A circuit may involve:

- heads
- MLP blocks
- features
- neuron groups
- residual-stream subspaces

A circuit does not have to be a tiny sparse list of parts.

It can be somewhat distributed.

## Family vs Circuit

A family is an analysis grouping.

A circuit is a mechanistic claim.

So:

- family = “these features move together”
- circuit = “these parts causally implement the behavior”

Families help us search for circuits, but they are not the same thing.

## Forward Pass

The forward pass is the actual computation the model performs on the input.

This is the computation used during inference and also during training before backprop.

## Gradient

A gradient tells us how a small change in a weight would change the loss.

Large gradient means that changing that weight would matter more for the current batch.

## Update

An update is the actual change applied to the weights after using the gradient.

This is what makes the model at checkpoint `t+1` different from checkpoint `t`.

## Accuracy

Accuracy means how often the model predicts the correct answer.

For this project, the most important metric is usually **answer accuracy**, not overall token accuracy.

That is because many tokens in the stream are easy format tokens or stochastic write tokens.

## Heldout / OOD

Heldout means examples that are close to training but withheld in a controlled way.

OOD means out-of-distribution examples that differ more structurally.

These help us see whether the model learned a robust mechanism or only a narrow solution.

## Probe

A probe is a simple measurement that asks whether some information is present in an activation.

Example:

- can we decode the answer from `layer_2_post_mlp`?

A probe is evidence, not proof.

## Ablation

Ablation means removing or zeroing out some component and seeing what happens.

Examples:

- ablate a head
- ablate an MLP block
- ablate a neuron group
- ablate a feature

If performance drops, that component was probably doing useful work.

## Patching

Patching means taking information from one example or checkpoint and inserting it into another.

Examples:

- patch a residual state from a later checkpoint into an earlier checkpoint
- patch a feature family from `14000` into `7500`

This is one of the most useful tools for causal analysis.

## Lineage

Lineage means tracing which upstream components most affect a feature or family.

Example questions:

- which heads strongly affect feature `55`?
- which MLP block writes most into this family?
- which neuron group influences this feature most?

## What Happens During Training, Roughly

At a high level:

1. early on, the model has weak partial computations
2. some partial computations help a little
3. that changes the gradients
4. those gradients reinforce compatible components
5. stronger submechanisms appear
6. those submechanisms support later ones
7. eventually a fuller circuit forms

This is the main dynamical picture we are trying to study.

## Zoom Levels We Use

From zoomed out to zoomed in:

1. behavior
2. residual stages
3. heads and MLP blocks
4. features
5. feature families
6. neuron groups
7. individual neurons

We do not start with single neurons because that is too detailed and too unstable to be the main organizing view.

## What Our Current Tools Are Doing

`checkpoint-sweep`

- analyzes saved model checkpoints
- computes behavior and mechanistic metrics over time

`birth-window-analyze`

- finds windows where important changes happen

`birth-window-compare`

- compares selected checkpoints with patching and ablations

`shared-feature-fit`

- fits one shared feature basis for a stage across multiple checkpoints

`feature-trajectory-sweep`

- tracks the same features over all checkpoints

`feature-birth-analyze`

- estimates when features become active or useful

`feature-compare`

- compares feature changes between two checkpoints

`feature-patch`

- tests causal effects of selected features

`feature-lineage`

- traces which heads, MLPs, and neuron groups affect selected features

`feature-family-cluster`

- groups features that evolve similarly over training

## What We Are Trying To Understand

Not just:

- what the final model does

But:

- how the mechanism appears
- which pieces appear first
- which pieces reinforce later pieces
- why some branches are strengthened and others are not
- whether this can be summarized by a low-dimensional dynamical description

## Important Caution

Not every analysis object is a “real natural unit”.

For example:

- a feature is an analysis object
- a family is an analysis grouping
- a circuit is a stronger mechanistic claim

So we should treat these tools as ways to search for structure, then validate that structure causally.
